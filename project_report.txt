=========================================================
          تقرير المشروع الشامل: QuickBite (كويك بايت)
=========================================================

1. نظرة عامة على المشروع (Project Overview)
---------------------------------------------------------
"كويك بايت" هو نظام إلكتروني متكامل (Full-Stack) لإدارة المطاعم وطلبات الطعام أونلاين. يهدف التطبيق إلى تقديم تجربة مستخدم سلسة وعصرية للعملاء لطلب الطعام، وتوفير لوحة تحكم (Dashboard) قوية واحترافية للإدارة لمراقبة الطلبات وتحديث قائمة الطعام بشكل فوري.


2. التقنيات والأدوات المستخدمة (Tech Stack)
---------------------------------------------------------
الواجهة الأمامية (Frontend):
- إطار العمل: Next.js (الإصدار 14) مع React.
- التصميم والتنسيق: Tailwind CSS (لبناء واجهات احترافية وسريعة).
- المؤثرات البصرية: Framer Motion (لإضافة حركات انتقالية وتفاعلية جذابة).
- تعدد اللغات: Next-Intl (لدعم اللغتين العربية والإنجليزية بشكل كامل).

الخلفية البرمجية (Backend):
- إطار العمل: FastAPI (لبرمجة واجهات برمجة تطبيقات API سريعة جداً باستخدام لغة Python).
- قواعد البيانات: SQLite للبيئة المحلية، وتم تهيئة الكود لدعم PostgreSQL في بيئة الإنتاج (Production).
- التعامل مع قواعد البيانات: SQLAlchemy (كنظام ORM).
- الأمان والمصادقة: JWT (JSON Web Tokens) لتسجيل الدخول وحماية الـ Routes وتحديد الصلاحيات (Admin vs User).


3. المميزات والخصائص المنفذة (Implemented Features)
---------------------------------------------------------
أ) تجربة العميل (Customer Experience):
- دعم كامل للغتين (Arabic/English) مع تغيير الاتجاهات تلقائياً (LTR إلى RTL).
- تصميم متجاوب (Responsive) متوافق مع كافة أحجام الشاشات، مع استخدام "Bottom Navigation Bar" في الهواتف الذكية.
- واجهة ترحيبية (Hero Section) عبارة عن عارض صور (Carousel) متحرك لعرض أشهى الأطباق وجذب العملاء.
- نظام سلة مشتريات (Cart Drawer) يفتح بسلاسة من الجانب ويظهر فوق كافة العناصر في الهواتف.
- صفحة الدفع (Checkout) تتيح اختيار طريقة الدفع وعناوين توصيل بديلة.
- صفحة تتبع الطلب (Track Order) مزودة بخط زمني (Timeline) تفاعلي يعرض حالة الطلب من لحظة استلامه حتى توصيله.

ب) لوحة الإدارة (Admin Dashboard):
- نظام إحصائيات يعرض (إجمالي الطلبات، الطلبات النشطة، والأرباح).
- إدارة قائمة الطعام (Menu Management): إمكانية إضافة، تعديل، إخفاء (Toggle Active)، أو حذف الوجبات عبر نافذة منبثقة (Modal).
- مراقبة وتحديث حالة الطلبات الحية (Pending, Preparing, Ready, Delivered) مع تغيير الحالة بنقرة واحدة.
- إدارة العملاء (Customers Management) للاطلاع على بيانات المسجلين في النظام.
- قائمة جانبية (Side Navigation Bar) ثابتة وذكية في التحكم في الصفحات.


4. التحديثات والإصلاحات الأخيرة التي تم تنفيذها (Recent Work)
---------------------------------------------------------
- مراجعة اللغات (i18n Audit): تم المرور على كافة الملفات وسحب جميع الكلمات الإنجليزية الثابتة (Hardcoded) وتحويلها إلى قواميس الـ JSON، مع ضبط كلاسات (RTL) لضمان عدم وجود أي عيوب في التصميم باللغة العربية.
- إصلاح تداخل العناصر (Z-Index Fix): حل مشكلة تغطية القائمة السفلية على أزرار سلة المشتريات لضمان أفضل تجربة مستخدم على الجوال.
- تهيئة الاستضافة (Deployment Preparation): 
   - فصل روابط الـ Localhost واستخدام متغيرات البيئة (`NEXT_PUBLIC_API_URL` و `FRONTEND_URL`).
   - تجهيز ملفات `requirements.txt` و `render.yaml` لرفع الباك-إند.
   - تهيئة الباك-إند للعمل مع قاعدة بيانات PostgreSQL.
- إصلاح أخطاء الـ TypeScript، والتأكد من توافق أنواع المتغيرات (Types).
- رفع الكود (Git Push): تم ربط المشروع بمستودع GitHub ودمج التغييرات ورفع الكود بنجاح تام بعد حل مشاكل المصادقة وتضارب الفروع (Rebase).


5. ملخص (Conclusion)
---------------------------------------------------------
المشروع الآن عبارة عن بيئة برمجية نظيفة، متجاوبة، مدعمة بلغتين، وجاهزة بنسبة 100% لعملية الاستضافة (Deployment) على منصات مثل Vercel (للفرونت إند) و Render (للباك إند).
